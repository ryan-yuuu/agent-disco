1"""Semantic key vocabulary over :mod:`readchar`. 

2 

3readchar owns the parts that are genuinely hard — reconfiguring the terminal, 

4restoring it on the way out, and parsing multi-byte escape sequences into one 

5string. (Not *raw* mode: it clears ICANON/ECHO from c_lflag and leaves c_iflag 

6alone, so input translation such as CR->LF stays on. That distinction matters — 

7see the CR note below.) This module owns only the mapping from those raw strings 

8onto the :class:`Key` vocabulary the widgets navigate by, plus the aliases 

9readchar does not cover for our use (``docs/design/cli-tui-migration.md`` §4.1). 

10 

11:func:`read_key` is the single input seam: widgets take it as an injectable, so 

12every widget is testable by feeding a scripted key list with no TTY. 

13""" 

14 

15from __future__ import annotations 

16 

17import errno 

18import sys 

19import termios 

20from enum import Enum, auto 

21 

22import readchar 

23 

24 

25class Key(Enum): 

26 """A control key a widget acts on. Printable input never becomes a ``Key``. 

27 

28 Space is deliberately absent. It is *printable text* — the character at the 

29 centre of every prose answer — and only the checkbox reads it as a command. 

30 Binding it here would make it a control key everywhere, which is exactly the 

31 bug that silently turned "npx -y pkg" into "npx-ypkg": the text field asks 

32 for literal input, sees a Key, and drops it. A widget that wants space as a 

33 command matches the raw character itself, the way ``confirm`` matches y/n. 

34 """ 

35 

36 UP = auto() 

37 DOWN = auto() 

38 ENTER = auto() 

39 BACKSPACE = auto() 

40 EOF = auto() 

41 

42 

43# Raw sequence -> Key. Some entries are hand-written literals rather than 

44# readchar constants, for three different reasons: 

45# 

46# * "\x1bOA"/"\x1bOB" are the application-cursor-mode (DECCKM) arrow forms. 

47# readchar defines only the normal-mode "\x1b[A"/"\x1b[B" forms, so a terminal 

48# that has switched modes would navigate nowhere without these. THIS is the 

49# load-bearing alias. 

50# * CR ("\r") maps to ENTER alongside LF. On POSIX this is belt-and-braces, NOT 

51# a correction: readchar is not in raw mode — it clears only c_lflag bits 

52# (ICANON/ECHO) and never touches ICRNL in c_iflag — so the tty driver still 

53# translates CR to LF and ``readchar.key.ENTER`` ("\n") matches on its own. 

54# The "\r" binding covers the cases where that is not true (Windows, or a 

55# terminal already in true raw mode). An earlier version of this comment 

56# claimed Enter would be "dead" without it; that was inferred from the 

57# termios call without checking which flag word it touched, and a pty probe 

58# disproved it. 

59# * "\x08" is Ctrl-H / the Windows backspace. readchar's BACKSPACE is "\x7f" on 

60# POSIX, so this is a genuinely separate binding, not a restatement. 

61# 

62# Esc is deliberately absent: ``readchar.readkey`` blocks after "\x1b" waiting to 

63# disambiguate an escape sequence, so a lone Esc press cannot be observed. Ctrl-C 

64# is the cancel key instead — readchar raises KeyboardInterrupt for it, which the 

65# CLI entry point already maps to "aborted." and exit 130. 

66_BINDINGS: dict[str, Key] = { 

67 readchar.key.UP: Key.UP, 

68 readchar.key.DOWN: Key.DOWN, 

69 "\x1bOA": Key.UP, 

70 "\x1bOB": Key.DOWN, 

71 "\r": Key.ENTER, 

72 "\n": Key.ENTER, 

73 readchar.key.BACKSPACE: Key.BACKSPACE, 

74 "\x08": Key.BACKSPACE, 

75 readchar.key.CTRL_D: Key.EOF, 

76} 

77 

78 

79def resolve(raw: str) -> Key | None: 

80 """Map a raw readchar string onto a :class:`Key`, or ``None`` if unbound. 

81 

82 ``None`` means **"not a bound key"** — which is a superset of text, not a 

83 synonym for it. Tab, Ctrl-A, Home and the left/right arrows all return 

84 ``None`` while being unprintable, so a caller must still ask whether ``raw`` 

85 is printable before treating it as input. A caller that skips that check 

86 injects raw escape sequences into the value. 

87 

88 (An earlier version of this docstring said ``None`` meant the caller could 

89 treat ``raw`` as literal input. That was wrong, and it is the same shape as 

90 the space bug: input filed under the wrong category. The check lives at 

91 ``widgets._typed_field``.) 

92 """ 

93 return _BINDINGS.get(raw)

96def read_key() -> str: 

97 """Block for one keypress and return its raw string. 

98 

99 Delegates to :func:`readchar.readkey`, which handles raw mode and escape 

100 sequences and raises :exc:`KeyboardInterrupt` on Ctrl-C (which the CLI entry 

101 point maps to a clean "aborted." exit 130). This is the seam widgets inject 

102 in tests. 

103 

104 Everything below funnels a broken stdin into one ``OSError(ENOTTY)``, because 

105 the entry point's non-TTY handler catches ``OSError`` and nothing else — and 

106 a broken stdin breaks in three different ways, on three different lines: 

107 

108 * **fd 0 closed at exec** (a supervisor or daemon spawning us with no stdin): 

109 CPython sets ``sys.stdin`` to ``None``, and readchar reaches 

110 ``sys.stdin.fileno()`` *before* ``termios``, so this raises 

111 :exc:`AttributeError` — checked first, since the later guards never see it. 

112 * **piped / CI stdin**: ``termios.tcgetattr`` raises :exc:`termios.error`, 

113 which despite being an OS-level failure does **not** subclass ``OSError``. 

114 * **a closed file object**: ``ValueError: I/O operation on closed file``. 

115 

116 Only the second of those was handled at first, and the end-to-end test used 

117 ``DEVNULL`` — a real fd — so it exercised that one path and missed the rest. 

118 Each raw exception dumps a traceback at an operator whose only mistake was 

119 running an interactive command without a terminal. 

120 

121 ``ValueError`` is caught but :exc:`io.UnsupportedOperation` is deliberately 

122 NOT special-cased: it already subclasses ``OSError``, so it reaches the 

123 handler on its own. That is an accident of CPython's hierarchy rather than a 

124 guarantee, which is why the explicit guards exist around it. 

125 """ 

126 # Not a broad `except AttributeError` around readkey(): that would mask 

127 # genuine readchar bugs. The condition is knowable up front, so ask. 

128 if sys.stdin is None: 128 ↛ 129line 128 didn't jump to line 129 because the condition on line 128 was never true

129 raise OSError(errno.ENOTTY, "stdin is not available (fd 0 is closed)") 

130 try: 

131 return readchar.readkey() 

132 except termios.error as exc: 

133 raise OSError(errno.ENOTTY, "stdin is not an interactive terminal") from exc 

134 except ValueError as exc: 

135 raise OSError(errno.ENOTTY, "stdin is closed") from exc
