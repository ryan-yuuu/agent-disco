1"""Substrate lifecycle orchestration: ``start`` / ``stop`` / ``status`` (§13.1-§13.3). 

2 

3This is the imperative glue above the two pure-ish seams — :mod:`compose` (renders 

4the project) and :mod:`client` (the REST wire) — that brings the *office* up and 

5down. Everything that touches the world (the binary path, the process launcher, 

6the wall clock) is injected, so the whole flow is unit-testable with no real 

7``process-compose`` binary and no broker. 

8 

9The hard-won contract from the Phase-0 spike (design §13) lives here: 

10 

11* **Detached launch** is ``up -f <yaml> -D -t=false -p <port> -L <log>`` — never 

12 ``--no-server`` (that kills the REST API the readiness gate polls). The REST 

13 port is *derived from the home path* (:func:`pc_port_for`) so two installs on 

14 one host do not both grab the supervisor default :8080. 

15* **Priming reconcile for upstream bug #494** (§13.1): immediately after ``up``, 

16 issue exactly one no-op ``update_project`` with the byte-identical rendered 

17 YAML, so the buggy first project-update lands on a no-op instead of bouncing the 

18 substrate. 

19* **Readiness gate** (§12.6 / §13.3): ``up -D`` returning 0 does NOT mean healthy, 

20 so ``start`` polls the **bridge** to ``is_ready``/Running with a timeout and, on 

21 timeout, tears the substrate down and returns non-zero — a green light that 

22 lies is worse than a red one. 

23* **Lock + idempotency** (§12.4): an exclusive ``flock`` serializes start/stop so 

24 two concurrent starts cannot race two supervisors; ``start`` probes first and 

25 short-circuits if the office is already open, and ``stop`` is a no-op if nothing 

26 is running. 

27 

28Import-light like the rest of this package. 

29"""

187def _read_log_tail(path: str, *, max_bytes: int = _LOG_TAIL_BYTES) -> str | None: 

188 """Return the last ``max_bytes`` of ``path`` as text, or ``None`` if unreadable. 

189 

190 Bounded (seeks to the tail rather than reading the whole file) so a large or 

191 rotated log cannot blow up memory. A missing/unreadable file yields ``None`` 

192 instead of raising — diagnosis is best-effort and must never itself fail the 

193 caller. Bytes are decoded leniently (``errors="replace"``) since a truncated 

194 tail may slice a multibyte sequence. 

195 """ 

196 try: 

197 size = os.path.getsize(path) 

198 with open(path, "rb") as handle: 

199 if size > max_bytes: 

200 handle.seek(size - max_bytes) 

201 data = handle.read() 

202 except OSError: 

203 return None 

204 return data.decode("utf-8", errors="replace")

260def pc_port_for(home: str | os.PathLike[str]) -> int: 

261 """A deterministic Process Compose REST port derived from the home path. 

262 

263 Two ``$CALFCORD_HOME`` installs on one host must not both grab the supervisor 

264 default :8080 (§12.4). We hash the *absolute* home — so a relative invocation 

265 picks the same port as the absolute one — with a stable digest (NOT Python's 

266 per-process-salted ``hash()``) into a documented high band. Same home always 

267 yields the same port, across processes and reboots, so every REST call and the 

268 ``up -p`` flag agree. 

269 """ 

270 absolute = os.path.abspath(os.fspath(home)) 

271 digest = hashlib.sha256(absolute.encode("utf-8")).digest() 

272 offset = int.from_bytes(digest[:8], "big") % _PORT_RANGE_WIDTH 

273 return _PORT_RANGE_START + offset

280@contextlib.contextmanager 

281def lifecycle_lock(home: str | os.PathLike[str]): 

282 """Hold an exclusive ``flock`` over ``<home>/state/calfcord-lifecycle.lock``. 

283 

284 Serializes ``start``/``stop`` so two concurrent invocations cannot race two 

285 supervisors against one home (§12.4). Uses ``LOCK_EX | LOCK_NB`` so a second 

286 holder fails *immediately* instead of blocking indefinitely. CONTENTION 

287 surfaces as :class:`~calfcord.supervisor._workspace.WorkspaceBusyError` — a 

288 domain refusal ``start``/``stop`` translate into one clean error line (the 

289 holder may be another start/stop OR a roster verb holding the lock SHARED for 

290 its spawn-confirm window, so the message names neither). Any other ``OSError`` 

291 (permissions, a broken state dir) propagates as the IO problem it actually is. 

292 The parent ``state/`` dir is created on demand; the lock file itself is kept 

293 (its presence is harmless — the advisory lock, not the file, is the guard) 

294 and the fd is always closed on exit, releasing the lock. 

295 """ 

296 path = _lock_path(home) 

297 os.makedirs(os.path.dirname(path), exist_ok=True) 

298 fd = os.open(path, os.O_CREAT | os.O_RDWR, 0o644) 

299 try: 

300 try: 

301 fcntl.flock(fd, fcntl.LOCK_EX | fcntl.LOCK_NB) 

302 except BlockingIOError as exc: 

303 raise _workspace.WorkspaceBusyError( 

304 "another disco command is in progress for this workspace " 

305 f"(could not acquire {path}); retry in a moment." 

306 ) from exc 

307 try: 

308 yield 

309 finally: 

310 fcntl.flock(fd, fcntl.LOCK_UN) 

311 finally: 

312 os.close(fd)

332async def _supervisor_belongs_to_home( 

333 client: ProcessComposeClient, home: str 

334) -> bool | None: 

335 """Whether the answering supervisor was launched for ``home`` (Fix #11). 

336 

337 :func:`pc_port_for` maps two homes into an 800-port band, so a collision is 

338 possible: two installs can hash to one REST port. The bare ``project_state`` 

339 idempotency probe only proves *something* answers that port, not *whose* 

340 supervisor it is — so before trusting an "already up" verdict we read back a 

341 declared process's config (which embeds the home-specific log path) and check 

342 for this home's marker. 

343 

344 Returns ``True`` (this home's), ``False`` (a DIFFERENT home colliding on the 

345 port — the caller must fail loudly, never a false "already open"), or ``None`` 

346 when it cannot be determined (the info route is unavailable), in which case the 

347 caller keeps the prior best-effort idempotent behavior. 

348 """ 

349 try: 

350 # ``bridge`` is always a declared substrate process, so its config is the 

351 # stable place to read the home-specific log path back from. 

352 info = await client.get_process_info("bridge") 

353 except RuntimeError: 

354 return None 

355 if not info: 

356 return None 

357 # Robust to whatever JSON key Process Compose uses for the log path (the API 

358 # shape is version-fragile, §12.4 Risk #2): scan the whole serialized config 

359 # rather than pinning a single field name. Match the marker only where it 

360 # OPENS a quoted path value (repr quotes string values, and the absolute 

361 # home path starts the log_location/working_dir it appears in) so a home 

362 # whose path is a *suffix* of another's — e.g. "/calf" vs "/data/calf" — 

363 # cannot false-positive on a bare substring scan and silently adopt the 

364 # other install's colliding supervisor. 

365 marker = _home_marker(home) 

366 serialized = repr(info) 

367 return f"'{marker}" in serialized or f'"{marker}' in serialized

370def _bridge_is_ready(state: object) -> bool:

407async def _await_bridge_ready( 

408 client: ProcessComposeClient, 

409 *, 

410 timeout_s: float, 

411 clock: Clock, 

412 sleep: Sleep, 

413 restarted_from_pid: int | None = None, 

414) -> bool: 

415 """Poll the bridge until it is ready, bounded by ``timeout_s`` (§13.3). 

416 

417 A transport error mid-poll (the supervisor restarting the bridge under 

418 ``restart: always``) is treated as "not ready yet", not a fatal error, so a 

419 transient bounce does not abort the gate before the budget is spent. 

420 

421 ``restarted_from_pid`` gates the in-place restart path (:func:`_restart_bridge_to_ready`): 

422 when set, ``Ready`` is accepted only once the reported pid DIFFERS from it, so a 

423 poll that races Process Compose cannot latch the pre-restart instance's stale 

424 ``Ready`` and lie (§12.6 — the bridge is ``Ready`` right up until it bounces, and 

425 its heartbeat-freshness probe can keep reading ``Ready`` off the old beat for a 

426 beat). The cold-start path leaves it ``None`` — there is no prior instance, and 

427 the bridge begins not-``Ready`` there anyway. A restarted bridge that never 

428 reports a pid degrades to the plain readiness gate rather than hanging. 

429 """ 

430 deadline = clock() + timeout_s 

431 while True: 

432 try: 

433 state = await client.get_process("bridge") 

434 except RuntimeError: 

435 state = None 

436 if _bridge_is_ready(state) and (restarted_from_pid is None or _process_pid(state) != restarted_from_pid): 

437 return True 

438 if clock() >= deadline: 

439 return False 

440 await sleep(_READINESS_POLL_INTERVAL_SECONDS)

443async def _restart_bridge_to_ready( 

444 client: ProcessComposeClient, 

445 home: str, 

446 *, 

447 ready_timeout_s: float, 

448 clock: Clock, 

449 sleep: Sleep, 

450) -> str | None: 

451 """Restart the bridge slot in place and wait until the NEW instance is Ready. 

452 

453 The shared seam behind ``disco bridge restart`` and ``start``'s already-open 

454 path. Restarts ONLY the ``bridge`` process inside the running supervisor — 

455 never relaunches the supervisor, so the §12.4 "no second supervisor" invariant 

456 holds for both callers. 

457 

458 Returns ``None`` on success (the restarted bridge reached Ready). On failure it 

459 returns a ready-to-print operator message; the caller prints ``error: <msg>`` 

460 and returns non-zero. §12.6 honest fail-fast: the caller must NOT report success 

461 on a non-``None`` return. Two failure modes are distinguished because they need 

462 different guidance: 

463 

464 * the restart REST call itself failed (the bridge never bounced) — surface the 

465 :class:`ProcessComposeError` (its HTTP status + Process Compose's reason), the 

466 way ``stop`` echoes its teardown error, instead of a misleading "not ready"; 

467 * the bridge bounced but never reached Ready within ``ready_timeout_s`` — 

468 diagnose the cause from ``bridge.log`` (a rejected token / disabled intent / 

469 missing ``DISCORD_*`` field is the likely culprit exactly when a restart is 

470 picking up new config), reusing :func:`_diagnose_start_failure`, else a 

471 generic "check the log". 

472 

473 Readiness is gated on the reported pid CHANGING from the pre-restart value (via 

474 :func:`_await_bridge_ready`), so a poll racing Process Compose cannot latch the 

475 old instance's stale ``Ready``. The caller owns the lifecycle lock; this must 

476 NOT re-acquire it — the ``flock`` is non-reentrant, so a second acquire refuses. 

477 """ 

478 try: 

479 pid_before = _process_pid(await client.get_process("bridge")) 

480 except RuntimeError: 

481 pid_before = None # can't read it (e.g. slot not declared yet) → plain readiness gate 

482 try: 

483 await client.restart_process("bridge") 

484 except RuntimeError as exc: 

485 # The restart request itself failed — the bridge never bounced, so "not 

486 # ready — check the bridge log" would misdirect. Surface the actual cause 

487 # (HTTP status + PC's reason) the way stop() echoes its teardown error. 

488 return f"the bridge restart request failed ({exc})." 

489 if await _await_bridge_ready( 

490 client, timeout_s=ready_timeout_s, clock=clock, sleep=sleep, restarted_from_pid=pid_before 

491 ): 

492 return None 

493 # Bounced but never became Ready: prefer the cause-specific diagnosis the 

494 # cold-start path gives (bad token / disabled intent / bad DISCORD_* field) — 

495 # the restart path is where a freshly-edited config most often breaks. 

496 diagnosis = _diagnose_start_failure(os.path.join(home, "state", "logs")) 

497 return diagnosis or "the bridge didn't come back ready — check `disco logs bridge`."

500def _default_spawn(argv: Sequence[str]) -> None: 

501 """Launch a detached child that outlives this process (the production spawn). 

502 

503 ``start_new_session=True`` puts the child in its own session so the supervisor 

504 keeps running after the CLI exits (and is not felled by a Ctrl-C delivered to 

505 the CLI's terminal group). stdout/stderr are discarded — the supervisor writes 

506 its own ``-L`` log file — so no pipe fills up and wedges the child. 

507 """

591async def start( 

592 home: str | os.PathLike[str], 

593 *, 

594 server_urls: str, 

595 launcher: str, 

596 ready_timeout_s: float = _DEFAULT_READY_TIMEOUT_SECONDS, 

597 client: ProcessComposeClient | None = None, 

598 spawn: Spawn | None = None, 

599 spawn_blocking: Spawn | None = None, 

600 clock: Clock | None = None, 

601 sleep: Sleep | None = None, 

602 broker_probe: BrokerProbe | None = None, 

603 banner: bool = True, 

604 reporter: StartReporter = NULL_REPORTER, 

605) -> int: 

606 """Open the workspace: render, launch detached, prime, gate on readiness. 

607 

608 Returns a POSIX exit code: ``0`` once the substrate (broker + bridge) is up 

609 and the bridge is ready; non-zero if the broker precondition fails fast, or 

610 (after tearing the substrate back down) if the bridge does not become ready 

611 within ``ready_timeout_s`` — never a green light that lies (§12.6). 

612 

613 An **external** broker is a **fast-fail precondition** (§13.2): before 

614 rendering or launching anything, ``start`` probes it via ``broker_probe`` 

615 (default derived from ``server_urls``); a down remote broker returns non-zero 

616 immediately with an actionable hint instead of burning the full 

617 bridge-readiness budget waiting for a bridge that can never connect. A 

618 **compose-managed** (loopback) broker is EXEMPT from this probe: it is a cold 

619 autostart process ``up`` itself launches (with its own readiness probe + 

620 ``depends_on`` graph), so pre-probing it would fast-fail the very broker being 

621 started. The same :func:`broker_is_compose_managed` predicate also drives 

622 whether the rendered manifest declares a local ``broker`` slot, so the two 

623 decisions cannot diverge. 

624 

625 ``banner`` governs the **terminal next-step signpost ONLY** — the closing 

626 "workspace open … -> disco agent start <name>" line (and its already-open and 

627 empty-org variants). Errors and warnings print regardless, on every branch: they 

628 are composed where their context lives, every caller wants them verbatim, and 

629 ``init``'s finish explicitly relies on ``start`` having already printed the 

630 specific cause. This is emphatically **not** a ``quiet`` flag; widening it to one 

631 would strand the wizard on a silent failure. 

632 

633 Pass ``banner=False`` when the caller is about to *take* the next step the 

634 signpost would name. ``disco start`` leaves it on — the operator is being handed 

635 back the prompt with a decision to make, and §12.6 requires that banner to always 

636 name what's next. But ``disco init`` and ``agent create``'s start-now go on to 

637 start the agent themselves, so for them the signpost tells the operator to run a 

638 command the very next line executes for them, and they narrate their own progress 

639 in their own voice besides. 

640 

641 ``reporter`` narrates the two waits that block a cold open — the REST server 

642 answering and the bridge reaching Ready — so the CLI can render live progress 

643 instead of leaving the operator staring at a silent terminal (§12.6's honesty rule 

644 applied to the *wait*, not just the verdict). It defaults to the silent 

645 :data:`~calfcord.supervisor._progress.NULL_REPORTER`, and a wait that fails is 

646 deliberately never marked done, so the reporter can name what actually hung. 

647 

648 It is **orthogonal to** ``banner``, and deliberately not the ``narrate`` callback 

649 ADR-0022 rejected. That callback would have re-routed prose this function already 

650 prints — for which a bool is the simpler, more legible answer, and ADR-0022 is 

651 right. The reporter re-routes nothing: these two waits print *nothing* on any 

652 branch, at any ``banner`` setting, and a bool cannot express "the bridge wait is 

653 in flight" to a spinner. ADR-0022's line still holds either way — signposts and 

654 outcomes are the caller's, causes are never — and the reporter emits stable ids 

655 rather than prose precisely so every word stays the caller's (ADR-0023). 

656 

657 

658 ``client`` / ``spawn`` / ``spawn_blocking`` / ``clock`` / ``sleep`` / 

659 ``broker_probe`` are injected for testing; in production they default to a 

660 per-home REST client, a detached subprocess spawner (``up`` must outlive the 

661 CLI), a blocking spawner (for the synchronous ``down`` teardown), 

662 ``time.monotonic``, ``asyncio.sleep``, and the real broker metadata probe. 

663 """ 

664 home = os.fspath(home) 

665 client = _resolve_client(client, home) 

666 spawn = spawn or _default_spawn 

667 spawn_blocking = spawn_blocking or _default_spawn_blocking 

668 clock = clock or time.monotonic 

669 sleep = sleep or asyncio.sleep 

670 # Read the defined roster once so the success banners can tell an empty org 

671 # from a defined one — an org with zero agents is steered to `agent create`, 

672 # not the pointless `agent start`. The banners are its ONLY consumer, and the 

673 # read lazily imports the heavy agents package, so a caller that suppressed the 

674 # signpost pays neither the import nor the filesystem scan. 

675 agents_defined = _agents_defined(home) if banner else False 

676 

677 with contextlib.ExitStack() as stack: 

678 # A contended lock (another start/stop, or a roster verb holding it SHARED 

679 # for its spawn-confirm window) is a domain refusal, not a traceback: one 

680 # clean error line, exit 1, and the holder is left undisturbed. 

681 try: 

682 stack.enter_context(lifecycle_lock(home)) 

683 except _workspace.WorkspaceBusyError as exc: 

684 print(f"error: {exc}") 

685 return 1 

686 # Idempotency (§12.4): if the office is already open, do NOT launch a 

687 # second supervisor. We DO cycle the bridge in place (below) so a re-run of 

688 # `start`/`init` picks up a new build/config and clears a wedged mesh reader 

689 # — restarting one slot within the existing supervisor keeps the "no second 

690 # supervisor" invariant intact. But the port can collide across homes (Fix 

691 # #11): verify the answering supervisor is THIS home's before touching it. A 

692 # DIFFERENT home colliding on the port must fail loudly — never a false 

693 # "already open", and never restart the other install's bridge.

701 "this one under a different $CALFCORD_HOME, then re-run " 

702 "`disco start`."

708 # retrying the bridge underneath.

710 client, home, ready_timeout_s=ready_timeout_s, clock=clock, sleep=sleep

715 if banner: 715 ↛ 723line 715 didn't jump to line 723 because the condition on line 715 was always true

721 "No agents defined yet -> disco agent create <name>"

725 # One predicate governs both the pre-launch probe and the manifest's broker 

726 # slot, so they can never diverge (§13.2). A loopback URL means the broker

728 # external URL means the operator's broker lives elsewhere. 

729 broker_managed = broker_is_compose_managed(server_urls)

732 # remote broker the bridge cannot reach Ready without it, so probe it

734 # instead of after the full bridge readiness timeout, and the workspace is

749 # The manifest is SUBSTRATE-ONLY: the roster (agents, ``tools``, 

750 # ``mcp-<server>``) spawns off Process Compose per slot, so nothing

756 )

859async def _sweep_roster_pidfiles(home: str) -> tuple[int, int]: 

860 """Terminate every live roster process and clear stale pidfiles; return 

861 ``(stopped, still_running)`` — the honest split of what actually died. 

862 

863 ``disco stop`` closes the WHOLE workspace, roster included — but the roster lives 

864 off Process Compose (Phase 2), so a ``down`` of the substrate leaves the detached 

865 agent/tools/mcp processes running. This sweeps ``state/run/*.pid``: a live-and-ours 

866 slot is terminated (SIGTERM→SIGKILL via :func:`_workspace.terminate_slot`, which 

867 also clears its pidfile), and a stale/torn/not-ours pidfile is swept without a 

868 signal. Run BEFORE the substrate ``down`` so agents can still publish their 

869 departure while the broker is up. 

870 

871 The survivor count comes straight from the terminate ENUM — both keep-the- 

872 pidfile outcomes leave a still-live pid behind: ``KILL_UNCONFIRMED`` (the 

873 reap window closed on it) and ``INDETERMINATE`` (identity unreadable, left 

874 untouched; ``terminate_slot`` already printed its per-slot warning). 

875 ``TERMINATED`` / ``KILLED`` are confirmed deaths (``process_was_stopped`` — 

876 the one membership definition). No liveness re-probe: the enum already 

877 carries the verdict, and a second probe could only disagree with it racily. 

878 An unreadable ``state/run`` propagates :class:`_workspace.SlotScanError` — 

879 the caller must NOT read that as "nothing to sweep". 

880 """ 

881 stopped = 0 

882 wedged = 0 

883 for slot, _pidfile in list(_workspace.iter_slot_pidfiles(home)): 

884 result = await _workspace.terminate_slot(home, slot) 

885 if result in (TerminateResult.KILL_UNCONFIRMED, TerminateResult.INDETERMINATE): 

886 wedged += 1 

887 elif result is not None and result.process_was_stopped: 

888 stopped += 1 

889 return stopped, wedged

892async def restart_bridge( 

893 home: str | os.PathLike[str], 

894 *, 

895 client: ProcessComposeClient | None = None,

907 * a DIFFERENT ``$CALFCORD_HOME`` install answers this home's REST port (a Fix

913

924 with lifecycle_lock(home): 

925 if not await _supervisor_is_up(client): 

926 print("error: the workspace isn't open — run `disco start` first.") 

927 return 1 

928 # A colliding port could answer for ANOTHER install; never restart its 

929 # bridge from here (Fix #11). ``None`` (info route unavailable) keeps the

933 f"error: another Agent Disco install is using REST port "

935 "Restart the bridge from that install's home." 

936 )

938 reason = await _restart_bridge_to_ready( 

939 client, home, ready_timeout_s=ready_timeout_s, clock=clock, sleep=sleep 

940 ) 

941 if reason is not None:

944 except _workspace.WorkspaceBusyError as exc: 

945 print(f"error: {exc}") 

946 return 1 

947 print("bridge restarted.")

951async def stop( 

952 home: str | os.PathLike[str], 

953 *, 

954 client: ProcessComposeClient | None = None, 

955 spawn_blocking: Spawn | None = None, 

956) -> int: 

957 """Close the WHOLE workspace — substrate AND roster; idempotent (§12.4). 

958 

959 Two teardowns under one lock: the roster is swept first 

960 (:func:`_sweep_roster_pidfiles` — terminate every live detached agent/tools/mcp 

961 process while the broker is still up so departures publish, and clear stale 

962 pidfiles), then the substrate is brought down. ``down`` is issued through the 

963 **blocking** seam so ``stop`` returns only after the supervisor has actually 

964 stopped — a fire-and-forget detached ``down`` would let a racing ``start`` collide 

965 with a supervisor still shutting down (§13.3). 

966 

967 A no-op reports honestly: with the substrate down AND nothing swept, "nothing to 

968 stop"; otherwise "workspace closed", noting how many roster processes were 

969 stopped — and, separately, any survivor (a wedged ``KILL_UNCONFIRMED`` or an 

970 unverifiable ``INDETERMINATE`` slot) so "closed" never silently overclaims. 

971 "Closed" is also VERIFIED: a ``down`` that 

972 raises (e.g. its bounded timeout) or a supervisor still answering afterwards 

973 reports "teardown may not have completed" and returns ``1``. An unreadable 

974 ``state/run`` aborts before touching anything (the sweep saw nothing — "closed" 

975 would strand every process behind it). A contended lifecycle lock (a roster 

976 verb mid-spawn, or another start/stop) is a clean one-line refusal. 

977 """ 

978 home = os.fspath(home) 

979 client = _resolve_client(client, home) 

980 spawn_blocking = spawn_blocking or _default_spawn_blocking 

981 

982 try: 

983 with lifecycle_lock(home): 

984 pc_up = await _supervisor_is_up(client) 

985 # Sweep the roster first (broker still up → clean departures), then the 

986 # substrate. The sweep also runs when the substrate is already down, so an 

987 # orphaned roster (a crashed supervisor) is still cleaned up by `disco stop`. 

988 try: 

989 terminated, wedged = await _sweep_roster_pidfiles(home) 

990 except _workspace.SlotScanError as exc: 

991 # The sweep saw NOTHING — claiming "closed" here would strand every 

992 # process behind the unreadable dir. Leave the world untouched. 

993 print(f"error: {exc}") 

994 print( 

995 "error: not stopping the workspace under unknown roster state — " 

996 "fix the state/run permissions and re-run `disco stop`." 

997 ) 

998 return 1 

999 teardown_uncertain = False 

1000 if pc_up: 

1001 binary = resolve_pc_binary() 

1002 try: 

1003 spawn_blocking([binary, "down", "-p", str(pc_port_for(home))]) 

1004 except Exception as exc: # incl. a wedged down's TimeoutExpired 

1005 print(f"error: workspace teardown failed ({exc}).") 

1006 teardown_uncertain = True 

1007 # `down` returning is not proof: re-probe before claiming closed. 

1008 if not teardown_uncertain and await _supervisor_is_up(client): 

1009 teardown_uncertain = True 

1010 except _workspace.WorkspaceBusyError as exc: 

1011 print(f"error: {exc}") 

1012 return 1 

1013 

1014 if teardown_uncertain: 

1015 print( 

1016 "error: teardown may not have completed — run `disco stop` again " 

1017 "or check `disco status`." 

1018 ) 

1019 return 1 

1020 if not pc_up and terminated == 0 and wedged == 0: 

1021 print("nothing to stop (workspace not running).") 

1022 return 0 

1023 if wedged: 

1024 print( 

1025 f"workspace closed ({terminated} roster process(es) stopped, " 

1026 f"{wedged} still running — see `disco logs`)." 

1027 ) 

1028 elif terminated: 

1029 print(f"workspace closed ({terminated} roster process(es) stopped).") 

1030 else: 

1031 print("workspace closed.") 

1032 return 0

1038 server_urls: str, 

1039 client: ProcessComposeClient | None = None, 

1040 probe: Callable[[str], Awaitable[list[str]]] | None = None, 

1041) -> int: 

1042 """Render a glanceable org board, or a "not running" hint (§12.6). 

1043 

1044 The **substrate** rows come from Process Compose (broker + bridge). The 

1045 **roster** rows are the Phase-2 reconciliation of two truths: this host's live 

1046 pidfiles (``state/run/*.pid``) and the broker-wide mesh presence (the same probe 

1047 ``agent_ps`` uses). For an AGENT slot the cross-product yields three states — 

1048 

1049 * pidfile-alive **and** mesh-registered → ``running``; 

1050 * pidfile-alive **only** → ``started, not registered (see disco logs)`` (up here 

1051 but not answering — just starting, or wedged); 

1052 * mesh-registered **only** (no local pidfile) → ``running (another host)``. 

1053 

1054 A slot whose pidfile names a DEAD process (a crash or clean exit — no 

1055 auto-respawn off PC) is rendered honestly as ``not running (exited — see 

1056 <log>)`` rather than omitted; ``disco stop``'s sweep is the acknowledge-and- 

1057 clear point that removes those files. The ``tools`` singleton and 

1058 ``mcp-<server>`` slots carry no mesh presence, so their pidfile is the whole 

1059 truth: live renders ``running``, dead renders the exited row. An unreadable 

1060 mesh roster view degrades to the pidfile-only view with a note (read-only 

1061 status must never crash). 

1062 

1063 ``probe`` / ``client`` are injected for testing. 

1064 """ 

1065 home = os.fspath(home) 

1066 client = _resolve_client(client, home) 

1067 

1068 if not await _supervisor_is_up(client): 

1069 print("workspace not running (start it with: disco start)") 

1070 # The detached roster outlives a dead supervisor — say so rather than 

1071 # implying the host is idle. 

1072 _workspace.note_local_survivors(home) 

1073 return 0 

1074 

1075 try: 

1076 payload = await client.list_processes() 

1077 except RuntimeError: 

1078 # The supervisor died between the up-probe above and this read; read-only 

1079 # status must degrade (the docstring's never-crash promise), and "no longer 

1080 # answering" is exactly the not-running state. 

1081 print("workspace not running (start it with: disco start)") 

1082 _workspace.note_local_survivors(home) 

1083 return 0 

1084 substrate = [p for p in _process_rows(payload) if p["name"] in _SUBSTRATE] 

1085 

1086 try: 

1087 # ONE snapshot feeds every roster set below (live/dead/unverifiable are 

1088 # projections of it): one scan, one identity read per slot — and a slot 

1089 # dying mid-render can never appear in two disagreeing sets. 

1090 verdicts = _workspace.classify_slots(home) 

1091 except _workspace.SlotScanError as exc: 

1092 # Unreadable state/run: the roster half of the board is UNKNOWN, not 

1093 # empty — warn and render what can still be read (the substrate). 

1094 print(f"warning: {exc}") 

1095 verdicts = {} 

1096 live = {s for s, v in verdicts.items() if v is procspawn.Identity.OURS} 

1097 dead = { 

1098 s 

1099 for s, v in verdicts.items() 

1100 if v not in (procspawn.Identity.OURS, procspawn.Identity.INDETERMINATE) 

1101 } 

1102 unverifiable = { 

1103 s for s, v in verdicts.items() if v is procspawn.Identity.INDETERMINATE 

1104 } 

1105 agent_slots = {slot for slot in live if _workspace.is_agent_slot(slot)} 

1106 other_slots = sorted(live - agent_slots) # tools + mcp-<server> 

1107 

1108 probe = _workspace.resolve_probe(probe) 

1109 mesh_unreadable = False 

1110 try: 

1111 # Mesh names are broker-wide input: screened (with one aggregate 

1112 # warning for any malformed name) before they can reach a printed row. 

1113 logical = _workspace.screen_mesh_names(await probe(server_urls)) 

1114 except Exception: 

1115 logical = set() 

1116 mesh_unreadable = True 

1117 

1118 print("workspace is open.") 

1119 print("substrate:") 

1120 for row in substrate: 

1121 print(_format_row(row)) 

1122 print("roster:") 

1123 _render_roster_board( 

1124 home=home, 

1125 agent_slots=agent_slots, 

1126 other_slots=other_slots, 

1127 dead_slots=dead, 

1128 unverifiable_slots=unverifiable, 

1129 logical=logical, 

1130 ) 

1131 if mesh_unreadable: 

1132 # Honest label: it is the mesh VIEW we could not read — the substrate 

1133 # rows above may well show the broker itself Running. 

1134 print("note: mesh roster view unreadable; roster shown from local pidfiles only.") 

1135 # Reboot non-survival, stated honestly (§12.6): the daemon is session-scoped. 

1136 print("note: the workspace does not survive a reboot; re-run `disco start`.") 

1137 return 0

1140def _render_roster_board( 

1141 *, 

1142 home: str, 

1143 agent_slots: set[str], 

1144 other_slots: list[str], 

1145 dead_slots: set[str], 

1146 unverifiable_slots: set[str], 

1147 logical: set[str], 

1148) -> None: 

1149 """Print the reconciled roster rows (see :func:`status`). 

1150 

1151 Agents reconcile pidfile-presence against mesh-presence into the three §3.4 

1152 states; ``tools`` / ``mcp-<server>`` slots have no mesh presence, so a live 

1153 pidfile renders ``running``. Dead-but-pidfile-present slots render the honest 

1154 ``not running (exited — see <log>)`` row instead of vanishing (crashes are the 

1155 board's whole job to surface under no-auto-respawn); an UNVERIFIABLE slot (a 

1156 live pid whose identity read failed) renders ``state unknown`` — neither the 

1157 ``running`` claim nor the ``exited`` lie. An entirely empty roster prints the 

1158 create/start signpost so the board is never a confusing blank. 

1159 """ 

1160 dead_agents = {slot for slot in dead_slots if _workspace.is_agent_slot(slot)} 

1161 dead_others = sorted(dead_slots - dead_agents) 

1162 unverifiable_agents = {slot for slot in unverifiable_slots if _workspace.is_agent_slot(slot)} 

1163 unverifiable_others = sorted(unverifiable_slots - unverifiable_agents) 

1164 agent_names = sorted(agent_slots | logical | dead_agents | unverifiable_agents) 

1165 if not agent_names and not other_slots and not dead_others and not unverifiable_others: 

1166 print(" (none running -> disco agent start <name>, or disco agent create <name> to add one)") 

1167 return 

1168 

1169 unverifiable_state = "state unknown (cannot verify its recorded process)" 

1170 for name in agent_names: 

1171 here = name in agent_slots 

1172 answering = name in logical 

1173 log_path = procspawn.log_path_for(home, name) 

1174 if here and answering: 

1175 state = "running" 

1176 elif here: 

1177 state = "started, not registered (see disco logs)" 

1178 elif name in unverifiable_agents: 

1179 state = unverifiable_state 

1180 elif answering and name in dead_agents: 

1181 # Both truths: the org still answers for the name elsewhere, but the 

1182 # local instance died and its pidfile is awaiting the stop-sweep. 

1183 state = f"running (another host); exited here — see {log_path}" 

1184 elif answering: 

1185 state = "running (another host)" 

1186 else: 

1187 state = f"not running (exited — see {log_path})" 

1188 print(f" {name:<16} {state}") 

1189 for slot in other_slots: 

1190 print(f" {slot:<16} running") 

1191 for slot in unverifiable_others: 1191 ↛ 1192line 1191 didn't jump to line 1192 because the loop on line 1191 never started

1192 print(f" {slot:<16} {unverifiable_state}") 

1193 for slot in dead_others: 

1194 print(f" {slot:<16} not running (exited — see {procspawn.log_path_for(home, slot)})") 

1195

1199 

1200 The wire-shape tolerance (bare list vs ``{"data": [...]}``, skip non-dicts) is 

1201 the one shared :func:`_workspace.iter_process_dicts` (Fix #14); this only 

1202 projects the board's three columns onto each entry. 

1203 """ 

1204 return [ 

1205 { 

1206 "name": item.get("name", "?"), 

1207 "status": item.get("status", "?"), 

1208 "is_ready": item.get("is_ready", "-"), 

1209 } 

1210 for item in iter_process_dicts(payload) 

1211 ]
