1"""The visual language, in one place. 

2 

3Deliberately **monochrome**: hierarchy comes from weight and dimming, never from 

4hue. Nothing here names a colour except :data:`ERROR`. 

5 

6Why no accent colour — including no off-white one. An off-white accent 

7(``#e6e6e6``) is only "off-white" on a dark terminal; on a light-background 

8profile it is near-invisible, and Solarized-style themes remap the 16-colour 

9palette out from under it. Bold-on-default-foreground is the one accent that 

10renders correctly on every terminal theme, because it asks the terminal for 

11emphasis instead of dictating a colour. So :data:`ACCENT` carries no hue. 

12 

13:data:`ERROR` is the single exception: red is a safety signal rather than an 

14accent, and it marks only genuine failures. It is one constant if we ever want 

15it gone too. 

16""" 

17 

18from __future__ import annotations 

19 

20from typing import Literal, get_args 

21 

22from rich import box 

23 

24# --- styles ----------------------------------------------------------------- 

25ACCENT = "bold" # the pointer, the row under the cursor, the focused field 

26MUTED = "dim" # descriptions, hints, secondary detail 

27BORDER = "dim" # panel edges — present, never loud 

28ERROR = "red" 

29 

30# "not dim" is load-bearing, not redundant. Rich renders a Panel's title inside 

31# the border row, so the title inherits ``border_style`` — and BORDER is dim. 

32# Without the explicit override the question comes out bold-and-dimmed: the most 

33# important text in the frame, washed out by its own frame. In a monochrome 

34# design, where weight carries all of the hierarchy, that is a real defect 

35# rather than a nicety. 

36TITLE = "bold not dim" 

37 

38# --- glyphs ----------------------------------------------------------------- 

39# The pointer is a heavy right-pointing angle ORNAMENT, not a plain greater-than. 

40# Ruff flags it as confusable; the distinction is the point. A greater-than reads 

41# as a quote marker or a shell prompt, where the ornament reads as a cursor — and 

42# it is the one glyph the whole selection language rests on. 

43POINTER = "❯" # noqa: RUF001 

44CHECK_ON = "◉" 

45CHECK_OFF = "○" 

46BULLET = "·" 

47 

48# The step-record vocabulary. TICK alone can only describe a flow that worked, and 

49# the flows that print records (`doctor`, `init`'s live finish) exist precisely to 

50# report honest partial success — a tools host that didn't come up, an agent that 

51# didn't register. WARN is "we could not confirm it", CROSS is "it is not so". 

52TICK = "✓" 

53WARN = "⚠" 

54CROSS = "✗" 

55 

56# The status a step record reports, and its glyph. Lives here — not in the flows that 

57# print one — because two boards each hard-coding "✓/⚠/✗" is two visual languages that 

58# agree only by luck: `doctor` and `init`'s live finish would drift the first time 

59# either gained a status. The map is the single source; the flows read it. 

60Status = Literal["ok", "warn", "fail"] 

61STEP_GLYPHS: dict[Status, str] = {"ok": TICK, "warn": WARN, "fail": CROSS} 

62# A typo'd status renders nothing and raises at the call site, so pin the map to the 

63# status domain at import (the idiom `doctor` established for exactly this map). 

64assert set(STEP_GLYPHS) == set(get_args(Status)), "STEP_GLYPHS drifted from Status" 

65 

66# Rounded edges read as modern and match the house style of the tools this CLI 

67# sits alongside. 

68BOX = box.ROUNDED 

69 

70# The spinner for a step slow enough to look hung. Braille dots read as motion at any 

71# font size and degrade to legible glyphs where the font lacks them. Its style must be 

72# named explicitly wherever it is used: Rich's default status spinner is GREEN, which 

73# would make it the one hued glyph in an otherwise monochrome CLI. 

74SPINNER = "dots" 

75 

76# The hint rendered in a widget's bottom border. Ctrl-C — not Esc — is the cancel 

77# key: readchar blocks after "\x1b" waiting to disambiguate an escape sequence, so 

78# a lone Esc press cannot be observed at all (design §4.1). Ctrl-C is also what the 

79# CLI already documents as safe and resumable, so this is honest rather than a 

80# compromise. 

81HINT_SELECT = f"↑↓ move {BULLET} enter select {BULLET} ctrl-c cancel" 

82HINT_CHECKBOX = f"↑↓ move {BULLET} space toggle {BULLET} enter confirm {BULLET} ctrl-c cancel" 

83HINT_TEXT = f"enter confirm {BULLET} ctrl-c cancel"
